## Decisions

1. **All-or-nothing import** returns 422 with `{row, column, error}` triples;
   the file's spreadsheet row numbers are used so a practitioner can fix the
   exact cells. Duplicate CINs *within the file* are an error (first
   occurrence wins the claim; the duplicate row is reported).
2. **Import of an existing CIN = update** (idempotent), restoring the row if
   it was soft-deleted (consistent with REVIEW F8 and the create endpoint).
   `unchanged` rows are counted and skipped — no noise in the audit log.
3. **Cap-table percentage mismatch is a warning, not an error** — cap tables
   are legitimately incomplete mid-entry (PRD §4.4 tolerance).
4. **No delete endpoints for directors/shareholders** — charter 10.6 forbids
   hard-deleting master data and the models have no soft-delete columns yet.
   Lifecycle is via cessation_date / is_active / share edits. Revisit in M4
   if the calendar needs director removal semantics.
5. Literal import/export routes live in a separate router registered before
   `/companies/{company_id}` so path matching stays unambiguous.
6. 5 MB upload cap (a 300-entity portfolio is well under 1 MB).
