## What was built

- **Loader → DB** (`python -m app.rules.load`): idempotent (unchanged payload
  inserts nothing); changed entries create a NEW version, never edit the old
  one; stale calendar rows are flagged immediately with an audit entry.
- **Calendar service**: attribute bag from company columns + per-FY facts
  (`company_fy_attributes`, incl. `fy-1` for CSR-style predicates); evaluator
  drafts persisted with upsert-by-(rule, subject, occurrence); user state
  (status, assignee, SRN, remarks, overrides) always survives regenerate;
  per-director expansion for `subject: director` rules (no DIN → skipped,
  DIN without allocation date → flagged, never guessed).
- **Routes**: generate (Executive+), list with full trace (rule_code, version,
  citation, effective_due_date = override > extension > computed),
  needs_review filter (the review queue), PATCH with the RBAC matrix rows —
  Executive assigned-rows-only, overrides Manager+ with mandatory reason,
  filed requires SRN or offline ack; Excel export; fy-attributes GET/PUT
  (Manager+ per Amendment A1).
- **Extensions**: recorded beside the computed date (never over it), applied
  from `rule_extension` rows at generate time, shown with circular reference.

## Decisions

1. Version-bump reconciliation matches existing rows by RULE CODE (not
   version id), so a new version revises its predecessor row instead of
   duplicating it.
2. A changed computed date on regenerate (rule change or master-data change,
   e.g. AGM moved) always flags `rule_revised` — one honest "date revised —
   review" signal rather than two subtly different ones.
3. `acknowledge_review` on PATCH clears the flag once a human has looked —
   the review queue is workable end-to-end.
4. Rows whose rule stops applying after an attribute change are NOT deleted
   (nothing is ever silently destroyed); they persist until reviewed. A
   "stale row" sweep is listed for M7 hardening.
5. Loader-triggered flagging runs synchronously; the charter's arq background
   job lands in M5 with the worker infrastructure.
