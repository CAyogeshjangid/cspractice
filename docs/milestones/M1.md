# M1 (+ partial M2/M4) — Foundation, decisions, and honest status

Date: 11 July 2026

## What was built and VERIFIED (executed, not just written)

- **Fail-fast Settings** (config.py): app refuses to start on missing vars,
  wildcard CORS, or placeholder secrets — unit-tested.
- **Security middleware**: headers with nonce-CSP and no unsafe-eval (C9),
  explicit-origin CORS (C1), unconditional CSRF double-submit with HMAC (C3),
  Redis fixed-window rate limiting, fail-closed on auth paths (C4).
- **Tenancy/auth**: register-first-user-becomes-Partner, argon2, JWT
  access+refresh in httpOnly cookies, RBAC dependency (`require_role`).
- **Rules engine core** (`app/rules/`): tri-state Kleene predicates with
  `fy-1` addressing, ordered anchor fallbacks, offset/fixed/occurrence date
  math, variants, supersession, extensions-beside-dates, phase gating —
  100% of engine behaviors covered by 35 passing unit + security tests.
- **Loader** refuses unsigned rules outside tests (charter §6).
- **Initial migration** (18 tables) autogenerated against a real Postgres 16,
  then hand-extended with: partial unique index `uq_company_firm_cin`
  (soft-delete-aware CIN uniqueness, REVIEW F8) and conditional
  INSERT-only grants for `praxis_app` on activity_log + immutable
  rule_version (C11). **Verified live**: UPDATE/DELETE denied, INSERT allowed.
- **End-to-end smoke against real Postgres+Redis**: CSRF 403 without token →
  register → create company → list with X-Total-Count; second firm gets 404
  on the first firm's company id and an empty list (tenant isolation);
  activity_log captured register/create with IP.
- **Frontend**: React 19 + Vite + TS strict scaffold with CSRF-aware client;
  typechecks and builds clean.
- **CI**: ruff + pytest (with real Postgres/Redis services) + alembic upgrade
  + gitleaks + frontend build, path-scoped to `praxis/**`.

## Decisions

1. **Lives under `praxis/` in nycaglobal for now** — session constraint.
   Extract with `git subtree split -P praxis` when the dedicated repo exists.
2. **Amendment A1 adopted** into CLAUDE.md: tri-state applicability,
   `company_fy_attributes`, anchor fallback lists, supersedes, phase,
   subject_type/subject_id, occurrences, FY conventions.
3. Rate limiter fails CLOSED for `/api/v1/auth/*` when Redis is down, OPEN
   elsewhere (credential-stuffing protection over availability).
4. Registration duplicate-email returns an opaque 409 (no user enumeration).
5. Re-import/re-create of a soft-deleted CIN restores and updates the row
   with a `restore` audit entry (REVIEW F8).
