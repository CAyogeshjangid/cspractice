## Decisions

1. **Refresh reuse → plain 401**, not automatic revoke-all-sessions. Detection
   of replay is inherent (jti consumed); full-family revocation deferred until
   there's a session-management UI to explain it.
2. **TOTP secret returned once at setup** alongside the otpauth URI so the
   frontend can render a QR client-side; secret lives only in Redis (10 min)
   until proven, then on the user row.
3. Invitation accept returns **410 Gone** for used/expired/unknown tokens —
   one opaque answer, no oracle.
4. Test personas are provisioned through the real invitation flow, so every
   RBAC test also exercises invitations end-to-end.
