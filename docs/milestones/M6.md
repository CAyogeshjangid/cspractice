## Decisions

1. **Templates ship UNSTAMPED.** `scripts/build_templates.py` (one-off, never
   imported by app code — C6) drafts the six skeletons in-house from bare
   statutory headings (S.101/134/118/170/88, SS-1/SS-2); no competitor
   content (charter 10.7). manifest.json carries no stamps; production use
   requires a named professional's stamp via the API. Bumping a template's
   version in the manifest CLEARS the old stamp — revalidation per version.
2. **StrictUndefined rendering.** Missing merge fields fail with a 422
   listing exactly the missing variables — a silent blank in a statutory
   document is a liability, not a default.
3. **Computed context always wins over caller params** — master data
   (company name, CIN, directors, shareholders) cannot be spoofed
   per-request; params only fill meeting details, business items, financial
   figures (Directors' Report figures are manual in v1 per PRD §4.7).
4. Generated files live under `STORAGE_DIR/{firm_id}/{company_id}/` (named
   volume in prod compose); download and snapshot endpoints are firm-scoped
   (cross-tenant 404s tested).
5. Template registry syncs from manifest.json via
   `python -m app.services.documents` (idempotent; never touches stamps).
